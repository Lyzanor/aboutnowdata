Explanation

A memory system built for humans first, and agents second.

About Now Data is not an assistant feature. It is a simple operational model for keeping identity, current priorities, and knowledge separate enough to remain legible.

About carries stable context: values, rules, identity, voice, boundaries. Now carries what matters today: active priorities, recent context, next steps. Data carries structured detail: notes, projects, references, tools.

The point is not to infer memory better. The point is to make memory inspectable, editable, portable, and safe for agents to read before they act.

Don't make humans learn the system. Make the system understand the human.

Base unit
Semantic files and folders: About, Now, Data.
Governance
Human-curated, explicit, and inspectable. No silent consolidation.
Temporal model
Clear separation between what is stable and what is current.
Main risk
The system becomes thin or stale if the user never updates it.
Main strength
Auditability and human legibility, even when multiple agents are involved.

About Skill

A skill to turn scattered notes into your own About.md.

Most people already have the raw material for an About: values, preferences, goals, voice notes, old bios, personal rules, and fragments from previous prompts. The skill should turn that messy input into a clear first draft that still feels authored by the person.

Give it

Fragments, not perfection.

  • Short notes about identity, work, style, and boundaries.
  • Examples of decisions, principles, or recurring preferences.
  • Existing docs that already describe the person imperfectly.

It returns

A structured draft you can inspect.

  • A readable About.md with identity, direction, and rules separated cleanly.
  • Language that stays explicit instead of sounding like generic assistant copy.
  • Open gaps marked clearly so the human can refine them fast.

Why it matters

Starting is the hard part.

  • Without a draft, most people never create the stable layer at all.
  • With a draft, the job becomes editorial: keep, remove, sharpen, own.
  • The result is a durable base any agent can read before acting.

Why This Shape

The memory model is the product.

Most memory systems optimize for automatic capture, latent inference, and silent consolidation. That feels convenient until the stored memory is wrong, outdated, or impossible to inspect.

About Now Data takes the opposite position. Memory should be legible. Stable context and changing context should never be mixed. Human control matters more than hidden cleverness.

Core rule

Agents should read before they act.

Agent Use

How an agent should use it.

1

Read About first

Start with identity, values, rules, and boundaries so behavior is aligned before any action begins.

2

Act from Now

Use the current state to prioritize correctly and avoid acting from stale assumptions.

3

Use Data for depth

Pull project details, notes, references, and supporting structure only when needed.

4

Write back explicitly

When the system changes, update the relevant file instead of inventing hidden memory elsewhere.

Why Now

Model advances make the current layer easy to maintain.

Thanks to recent improvements in the models themselves, refreshing Now is no longer the hard part. An agent can help summarize recent activity, extract next steps, or rewrite the current state in a few seconds, regardless of which model or ecosystem you use.

That makes About Now Data more than a local note-taking format. It becomes a transversal agent memory: one structure that different agents can read, update, and respect without depending on a single vendor or product.

The risk is not that agents cannot update the system. The risk is that the human never keeps it alive. The design goal is to make updating Now cheap enough that the system stays useful.

Now Skill

A skill to refresh Now.md from what actually changed this week.

The current layer should be cheap to update. A dedicated skill can review recent conversations, tasks, project notes, and calendar signals, then rewrite Now.md so priorities, risks, and next steps stay current without forcing the user to start from zero every time.

Reads from

Recent activity and momentum.

  • Last conversations, task lists, working notes, and recent project updates.
  • Signals about what moved, what stalled, and what became urgent.
  • Existing Now.md so continuity is preserved instead of reset.

Updates

A current layer that stays operational.

  • Top of mind, active projects, next steps, risks, and opportunities.
  • Clear wording that reflects the latest state, not generic summaries.
  • Minimal edits to keep the file light enough to maintain continuously.

Why it matters

If Now goes stale, the system drifts.

  • The biggest failure mode is not complexity. It is neglect.
  • When updates are cheap, the system remains useful across agents and tools.
  • That is what turns the format into living context instead of archived notes.

Adoption

Start smaller than you think.

You do not need a perfect personal operating system to begin. A short About.md, a minimal Now.md, and an empty Data/ folder are enough.

Keep About rare, keep Now easy, and let Data grow only where it proves useful. The system should remain valuable even if there is no automation at all.
